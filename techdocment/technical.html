Introduction

It is a web based comparison application where a user can do comparison of five different cloud service providers on basis of storage and there pricing, two type of price comparison can be done by using this application

  • Pay per use
  • Pas as you go
  • Price in detail comparison

    • Dynamic fetching is also done the data table in the database regarding to prices of cloud service providers are updated by searching the data on the service provider’s website at runtime. Price comparison regarding to different cloud service provider will be performed and best cloud service provider will be suggested to the user at the top. And also SLA’s (service level agreement) of different cloud service providers are available on our website.

    Requirement and Analysis

    Introduction

    In this chapter it will discuss complete detail of our project and which format I have used to complete the documentation of my project; here in this chapter I will also explain functional and non-functional requirements, design and implementation constraints of my project.

    Product Scope

    This is a web based comparison application which will compare cloud service providers on basis of their storage pricing on individual and enterprise level. Application will suggest best cloud service provider to user. Comparison can be done on the basis of both storage and price.

    The purpose of this application is to provide user best cloud service provider as there are too many cloud service providers in the market but people do not know which one is the best and which will serve them according to their needs and requirements. To overcome this hurdle and to make their selection easy we are developing this application.

    Software Design

    Use case Diagram

    An Actor is the major part of use case diagram because actor is an entity who is interacting with the system, an actor can be human, system or machine. In our use case diagram “user” is the actor and he is interacting with system, this user is a primary actor. This user is primary actor because he is asking for the services from system, user is having goal to do comparison of different cloud service providers so our system is providing service to the user and our system will fulfill the user requirement because it is designed to provide service to user.

    Class Diagram

    A class diagram is the collection of classes which tells the relationship and source code dependencies among the classes, a class in a class diagram tells us the methods and objects used in a class. It shows us the complete structure of system that how system works in different situation. It also tells us the relationship among the objects. There are three portions of a class in class diagram, first one is containing the name of the class and in second is having the attributes of that specific class and third one contains the operations which a class can execute.
